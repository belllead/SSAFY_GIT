1) 변수에 대한 설명으로 맞지 않는 것을 고르시오

1. 대소문자를 구분한다.
2. 한글을 이용해서 이름을 만들 수 있다.
3. 변수 이름에 $나 #등의 특수문자를 사용할 수 있다.
4. 합성어의 경우 camelCase를 따른다.



2) 다음 중 세부 타입이 다른 데이터형은?

1. byte
2. short
3. int
4. float



3) 다음 중 연산자의 종류가 다른 것은?

1. ==
2. >=
3. +=
4. !=


4) 객체지향 언어의 특징으로 옳지 않은 것은?

1. Abstraction(추상화)
2. Polymorphism(다형성)
3. Inheritance(상속)
4. Clarity(명확성)



1) 해설
변수 이름에는 $와 _를 제외한 특수문자는 허용되지 않는다.

2) 해설
byte, short, int, long은 정수형이고 float, double은 실수형이다.

3) 해설
+= 는 복합 대입 연산자이고, 다른 연산자는 비교 연산자이다.

4) 해설
객체지향 프로그램의 특징은 추상화, 다형성, 상속, 캡슐화이다.
명확성은 절차지향 언어의 특징이다.
